VsCode使用过程的相关整理

Visual Studio Code是一个轻量级但功能强大的源代码编辑器，可用于Windows，macOS和Linux。它内置了对JavaScript，TypeScript和Node的支持.js并具有针对其他语言和运行时（如C++，C#，Java，Python，PHP，Go，.NET）的丰富扩展生态系统

下载安装

推荐官网下载安装
官网：https://code.visualstudio.com/
下载镜像网站：vscode.cdn.azure.cn

在官方点击下载后，下载速度如果很慢，点击复制下载链接，将下载链接的地址中的 az764295.vo.msecnd.net 替换成 vscode.cdn.azure.cn 之后输入URL下载安装

登录账号

可以选择登录微软账号或者GitHub账号进行配置同步和拓展同步。之后在其他电脑登录后就可以直接同步，不需要重新搜索和安装拓展插件了

插件推荐

Chinese 汉化VsCode界面
GitLens 点击当前代码，可以知道当前所选这行代码是谁写的

各类配置

点击左下角设置按钮，可以通过设置编辑器字体样式大小，可以通过设置的主题的文件配置进行自定义修改

settings.json
{
    "editor.fontSize": 13,
    "workbench.preferredDarkColorTheme": "Default Light+",
    "workbench.colorCustomizations": { //覆盖默认配置
        "editor.background": "#353c45", //编辑器背景色
        // "editor.selectionBackground": "#5a5f21", //用户选中代码段的颜色 
        // "editor.findMatchBackground": "#ff0000", //当前搜索匹配的颜色
        // "editor.findMatchHighlightBackground": "#fa0a01", //其他搜索匹配项的颜色
        // "editor.findRangeHighlightBackground": "#ff9900", //限制搜索范围的颜色
        // "editor.lineHighlightBackground": "#3a2f2f", //光标所在行高亮内容的背景颜色
        // "editor.lineHighlightBorder": "#f33f3f" //光标所在行四周边框的背景颜色
    }
}

问题集合

使用终端无法执行Yarn命令

yarn : 无法加载文件 E:\Node\node_global\yarn.ps1，因为在此系统上禁止运行脚本。

在搜索栏输入 powershell 使用管理员身份打开
之后在窗口输入 set-ExecutionPolicy RemoteSigned 同时同意，输入 Y

VsCode 文件右侧的M标识是什么？

A: 你本地新增的文件（服务器上没有）.
C: 文件的一个新拷贝.
D: 你本地删除的文件（服务器上还在）.
M: 文件的内容或者mode被修改了.
R: 文件名被修改了。
T: 文件的类型被修改了。
U: 文件没有被合并(你需要完成合并才能进行提交)。
X: 未知状态(很可能是遇到git的bug了，你可以向git提交bug report)

关于 i18n ally key missing 插件的 文案路径不存在

明明项目在执行的时候是正确显示的，但是在VScode中就是有问题。

解决方案：

找到当前项目中 .vscode/settings.json 文件，没有就新建一个，修改路径。

{
  "i18n-ally.localesPaths": ["src/language"],
  "i18n-ally.enabledParsers": [ "json"], 
  "i18n-ally.keystyle": "nested", // 翻译路径格式,
  "i18n-ally.sourceLanguage": "en", // 翻译源语言
  "i18n-ally.displayLanguage": "zh-CN", //显示语言
}
