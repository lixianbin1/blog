CMD终端各类报错的相关整理

CMD终端各类报错的相关整理

报错

Electron打印在终端的中文变乱码

是因为windows控制台默认采用的是GBK编码，而我们的项目一般用的是UTF-8编码，所以日志输出到控制台会产生乱码。我们修改控制台的编码即可;控制台输入chcp 可以查看到编码未为 936 说明编码为GBK编码.

  • 临时

想解决当前窗口,则控制台修改编码:

控制台输入chcp 65001

  • 永久（需要管理员权限）

想解决所有窗口,则打开 window设置（右键点击开始菜单），选择时间和语言 - 管理语言设置 - 更改系统区域设置 勾选UTF-8 点击确定就可以了

Git 控制台中文名称显示乱码

当控制路径中的字符大于 0x80 的时候，转义路径字符，将编码显示设置为 false ，将不转义；

// 运行以下命令
git config --global core.quotepath false
